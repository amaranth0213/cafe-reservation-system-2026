# カフェ予約システム セットアップ手順書

## 1. Node.jsのインストール（まだの場合）

1. https://nodejs.org/ja/ にアクセス
2. 「LTS（推奨版）」をダウンロード・インストール
3. ターミナルで `node --version` と入力して `v20.x.x` と表示されれば完了

---

## 2. Supabaseのセットアップ（無料）

### アカウント作成
1. https://supabase.com にアクセス
2. 「Start your project」→ GitHubアカウントでサインアップ

### プロジェクト作成
1. 「New Project」をクリック
2. プロジェクト名：`cafe-reservation`（任意）
3. データベースパスワードを設定（メモしておく）
4. リージョン：`Northeast Asia (Tokyo)` を選択
5. 「Create new project」→ 1〜2分待つ

### データベーステーブルの作成
1. 左メニューの「SQL Editor」をクリック
2. 「New query」をクリック
3. `supabase/migrations/001_initial.sql` の内容をすべてコピー＆ペーストして「Run」
4. 同様に `supabase/migrations/002_generate_business_days.sql` も実行

### APIキーの取得
1. 左メニューの「Settings」→「API」
2. 以下をメモ：
   - `Project URL` → `NEXT_PUBLIC_SUPABASE_URL`
   - `anon public` → `NEXT_PUBLIC_SUPABASE_ANON_KEY`  
   - `service_role secret` → `SUPABASE_SERVICE_ROLE_KEY`（表示ボタンをクリック）

## 3. 管理者パスワードの設定

ターミナルで以下を実行（一度だけ）:

```bash
cd /Users/haru/カフェ予約システム作成
npm install
node -e "const bcrypt = require('bcryptjs'); console.log(bcrypt.hashSync('あなたのパスワード', 10));"
```

出力された文字列（`$2b$10$...`）をメモしておく。

---

## 4. 環境変数の設定

プロジェクトフォルダに `.env.local` ファイルを作成:

```bash
cp .env.local.example .env.local
```

`.env.local` を開いて以下を入力:

```
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co  ← 手順2でメモしたもの
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJh...               ← 手順2でメモしたもの
SUPABASE_SERVICE_ROLE_KEY=eyJh...                   ← 手順2でメモしたもの
ADMIN_PASSWORD_HASH=\$2b\$10\$...                   ← 手順3でメモしたもの（$を\$に置換）
```

⚠️ **重要**: `ADMIN_PASSWORD_HASH` の値に含まれる `$` は必ず `\$` に置換してください。
例: `$2b$10$abc...` → `\$2b\$10\$abc...`

## 6. Vercelへのデプロイ（無料公開）

### GitHubリポジトリ作成
1. https://github.com にアクセス（アカウント作成が必要）
2. 「New repository」→ 名前: `cafe-reservation`
3. 「Private」を選択して「Create repository」
4. ターミナルで:
```bash
cd /Users/haru/カフェ予約システム作成
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/あなたのユーザー名/cafe-reservation.git
git push -u origin main
```

### Vercelにデプロイ
1. https://vercel.com にアクセス（GitHubでサインアップ）
2. 「Add New Project」→ `cafe-reservation` リポジトリを選択
3. 「Import」をクリック
4. 「Environment Variables」に以下を追加:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `ADMIN_PASSWORD_HASH`
5. 「Deploy」をクリック → 2〜3分待つ
6. `https://cafe-reservation-xxxx.vercel.app` のようなURLが発行される

### インスタグラムへのリンク設定
発行されたURLの末尾に `/reserve` を追加した URL をインスタグラムのプロフィールリンクに設定:
```
https://cafe-reservation-xxxx.vercel.app/reserve
```

## 7. 毎週の営業日を追加する方法

### 自動的に追加（Vercel Cron設定）
Vercelの無料プランでも週1回のCronが使えます。

プロジェクトフォルダに `vercel.json` を作成:
```json
{
  "crons": [{
    "path": "/api/cron/generate-slots",
    "schedule": "0 0 * * 1"
  }]
}
```

### 手動で追加（Supabase SQLから）
Supabaseの「SQL Editor」で以下を実行:
```sql
SELECT generate_upcoming_mondays(8);
```

## 8. 管理画面の使い方

- **URL**: `あなたのサイト/admin/login`
- **ログイン**: 手順3で設定したパスワードを入力

### 予約一覧
- 日付でフィルター可能
- キャンセル処理はキャンセルボタンから

### メニュー管理
- 季節メニューの追加・削除
- 「表示中/非表示」で予約フォームへの表示を切り替え

### 営業日設定
- 各時間帯の「受付中止」ボタンで一時的に予約を止める
- 臨時休業日は「営業中/休業」ボタンで切り替え

### CSV出力
- 管理ダッシュボードの「CSVダウンロード」ボタン
- Excelで開けるファイル（文字化け防止済み）

## トラブルシューティング

### ページが表示されない
- `.env.local` の設定値が正しいか確認
- Supabaseプロジェクトが起動中か確認

### 予約できない
- Supabaseの「Table Editor」でテーブルが作成されているか確認
- SQLミグレーションが正常に実行されたか確認

### 管理画面にログインできない
- `ADMIN_PASSWORD_HASH` が正しく設定されているか確認
- bcryptハッシュの生成コマンドを再実行してみる
